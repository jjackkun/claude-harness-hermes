# 왜 스킬이 졌을까? — agent.md의 비밀

> 작성일: 2026-07-01
> 출처: 유튜브 "왜 스킬이 졌을까?(feat. agent.md 의 비밀)" / 채널 메이커 에반 (10:37)
> 원자료: Vercel(버셀)이 직접 실험·공개한 자료
> 목적: AI에게 최신 지식을 주입할 때 "스킬(Skill)"보다 "agent.md(상시 메모)"가 더 잘 동작한 이유와 실전 적용법 정리

## 개요

Vercel이 AI에게 일을 시키는 두 가지 방식을 동일한 난이도의 시험으로 비교했다.

- **스킬(Skill):** 필요할 때 AI가 꺼내 보는 전문 매뉴얼 묶음 (책장에 꽂힌 두꺼운 매뉴얼)
- **agent.md:** 프로젝트 폴더에 항상 펼쳐져 있는 텍스트 파일 하나 (책상 위에 항상 펴진 메모지)

상식적으로는 "필요할 때만 꺼내 보는" 스킬이 더 효율적·우월해 보이지만, 결과는 **agent.md 100점 vs 스킬 79점**으로 정반대였다. Vercel 보고서 표현: **"멍청한(단순한) 방식이 정교한 방식을 이겼다."**

## 1. 문제의 배경 — AI는 "공부를 몇 달 전에 끝낸 학생"

- AI는 학습(훈련) 시점 이후에 나온 새 기능을 본 적이 없다.
- 예: Next.js가 16버전으로 올라가며 새 기능이 대거 추가됨 → AI는 모르면서도 **옛날에 외운 방식**으로 코드를 짠다.
- Vercel의 과제: **새 문서(최신 지식)를 AI가 옛 기억 대신 보고 일하게 만드는 법**.

## 2. 실험 설계 — 4가지 상황, 똑같이 어려운 시험

AI가 한 번도 본 적 없는 새 기능만으로 시험지를 구성(가장 어려운 문제만).

| # | 상황 | 점수 |
|---|------|------|
| 1 | 멘붕 — 아무 도움말 없이 그냥 맡기기 | **53점** |
| 2 | 스킬을 주되 "알아서 써라" (자율) | **53점** |
| 3 | 스킬을 주고 "이거 꼭 써라" 강하게 지시 | **79점** |
| 4 | agent.md — 눈앞에 메모지 펼쳐두기 | **100점** |

- 4번은 빌드 100 / 검사(린트) 100 / 테스트 100 — **전 항목 만점**.
- agent.md 파일 하나가 가장 똑똑하다던 스킬을 **21점 차이로 승리**.

## 3. 왜 스킬은 실패했나

### 3-1. AI가 스킬을 "안 꺼내 본다"
- 스킬을 세팅해 둬도, 정작 그 상황이 오면 AI가 호출하지 않는다.
- 실험·발표자 경험 공통: **열에 여섯 번(약 60%)을 그냥 지나친다.**
- 결론(발표자): 자동 호출은 믿지 않고 **"지금 이 스킬 써"라고 직접 수동 호출**한다. 자동은 될 때도 안 될 때도 있지만, 수동은 100% 된다.

### 3-2. 스킬은 지나치게 예민하다 (지시문 한 단어에 결과가 휙휙)
- "무조건 매뉴얼부터 봐라" → AI가 매뉴얼만 보고 정작 프로젝트는 안 봄.
- "프로젝트 먼저 보고 그다음 매뉴얼 봐라" → 제대로 동작.
- **단어 순서 하나** 차이로 결과가 갈린다 → 지시를 아주 정확히 짜지 않으면 언제든 삐끗.

## 4. agent.md가 이긴 핵심 비결 — "AI에게 선택을 안 시킨다"

세 가지로 정리:

1. **고민할 순간이 없다.** 스킬은 "지금 매뉴얼을 볼까 말까"를 매번 고민 → 그 고민 지점이 곧 실수 지점. agent.md는 이미 눈앞에 있어 결정할 필요가 없다.
2. **항상 매 순간 거기 있다.** 스킬은 호출 타이밍을 놓칠 수 있지만, agent.md는 모든 작업 순간에 같이 있어 놓칠 수 없다.
3. **순서를 고민할 필요가 없다.** "매뉴얼 먼저? 코드 먼저?" 같은 순서 고민이 사라지고 자연스럽게 함께 본다.

> **핵심 원리:** 선택을 시키면 틀린다. 선택을 안 시키면 안 틀린다.
> 도구가 똑똑할수록 AI는 "쓸지 말지" 고민하다 삐끗한다. 능동적으로 찾아 쓰게 하지 말고, **가만히 있어도 보이게** 펼쳐주는 것이 AI를 다루는 핵심.

## 5. 한 단계 더 — 문서는 "압축해서, 목차만"

- 메모지를 펼쳐두는 건 좋지만, 문서가 너무 길면 책 한 권을 통째로 책상에 펴두는 격 → 일할 공간(컨텍스트)이 없어지고 오히려 헷갈린다.
- Vercel은 분량을 **약 40 → 8 수준으로, 약 80% 압축**.
- 전체 내용을 다 넣지 않고 **목차만** 삽입. "무슨 내용이 어디에 있는지"만 붙여 두고, AI가 필요할 때 해당 문서를 펴본다.
- 결과: **80%를 덜어내도 점수는 그대로 100점.** 핵심만 남겨 더 깔끔해졌다.

## 6. 스킬은 쓸모없나? — 역할 분담이 정답

| | agent.md | 스킬(Skill) |
|---|----------|-------------|
| 비유 | 매일 보는 **기본 안내문(메모지)** | 특별한 날 부르는 **전문가** |
| 용도 | 평소 기본 지식 — 항상 펼쳐 둠 | 내가 직접 시키는 특별한 큰 작업 — 필요할 때 호출 |

> 둘 중 하나만 쓰는 게 아니라, **평소 기본은 agent.md로 상시 노출 + 특별한 대형 작업은 스킬로 수동 호출**하는 분업이 정답.

## 7. 실전 적용법

### 7-1. Next.js 프로젝트 — 한 줄이면 끝
영상에서 보여준 명령(개념 표기):

```bash
npx next code mode 최신버전 agent.md
```

이 한 줄이 자동으로 3가지를 처리한다:
1. 내 프로젝트가 어떤 버전인지 확인
2. 그 버전에 맞는 문서를 내려받음
3. 그 문서의 **목차를 압축해서 agent.md에 자동 삽입**

> ⚠️ 위 명령은 영상의 설명용 표기다. 실제 적용 시 Next.js 공식 codemod의 정확한 명령어를 공식 문서에서 확인할 것.

### 7-2. Next.js를 안 써도 — 직접 만들기 (원리는 동일)
1. 프로젝트 **최상위 폴더**에 `agent.md` 파일을 하나 만든다.
2. AI가 꼭 알아야 할 것만 **목차처럼 짧게** 적는다. (길게 쓰지 말 것)
   - 예: "우리 프로젝트 규칙 — 색깔은 X, 폰트는 Y, 이 폴더엔 Z가 들어있다"
3. **문서 맨 위에 이 한 줄을 반드시 넣는다:**

   > 외운 지식 말고, 이 문서 먼저 보고 판단해.

   - AI는 자꾸 자기 옛 기억으로 일하려 하는데, 이 한 줄이 그것을 막아준다.
   - **"기억 말고 문서 봐"라는 이 한마디가 100점과 53점을 가른다.**

## 8. 결론 — 본질은 "파일 하나"가 아니라 "AI를 다루는 법"

- AI를 똑똑하게 만들려고 애쓰지 말고, **헷갈릴 틈을 안 주는 것**.
- **선택을 줄이고, 핵심만 압축해, 눈앞에 펼쳐주는 것.**
- 이 감각을 익히면 AI를 다루는 실력이 크게 달라진다.
